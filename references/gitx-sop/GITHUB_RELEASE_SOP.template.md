# GitHub Release SOP — {{PROJECT}}

> **Generated by** `gitx-sop` (gitx-release {{GITX_VERSION}}) on {{DATE}}.
> **Trigger**: dev says `推 GitHub` / `publish vX.Y.Z` → the project's AI follows
> these **8 Phases** in order.
>
> **Principles**
> - `gitx-release` produces release artifacts into `Release/{{PROJECT}}-vX.Y.Z/` (untouched).
> - This SOP only takes over "push the release to GitHub".
> - One independent commit + tag per version — clean snapshot, no WIP history.
> - The private Git host **never** appears in any GitHub metadata (URL / author / config).
> - **This document is a runbook for a human-supervised AI. It does NOT auto-run.
>   Every `git push` / `gh release` requires explicit human confirmation.**

## 0. Repo topology

| Layer | Location | Purpose | git |
|----|------|------|----------|
| Private main | project root | dev + HANDOFF + tests | private remote (no public exposure) |
| Publish worktree | `<root>/.github-publish-wt/` | GitHub snapshot (per-release rewrite) | **own .git**, origin = `https://github.com/{{REPO}}.git` |
| Release artifacts | `<root>/Release/{{PROJECT}}-vX.Y.Z/` | gitx-release output (.skill + tarball + SBOM) | not in git |

**Key facts**
1. GitHub source-of-truth = the sanitized public snapshot (Phase 4 redaction MUST pass).
2. Publish worktree is wiped + rebuilt every release (no cross-version history).
3. The private remote is **never** added to the publish worktree's `.git/config`.

## Phase 1. Pre-flight (all must pass before Phase 2)

```bash
cd "<project-root>"

# 1.1 gitx-release artifacts exist
[ -d "$RELEASE_DIR" ] && ls "$RELEASE_DIR"/{{PROJECT}}-${VER}.skill >/dev/null 2>&1 \
    && echo "✅ artifacts OK" \
    || { echo "❌ run /gitx-release ${VER} first"; exit 1; }

# 1.2 git tag exists (gitx-release tagged it)
git tag --list | grep -qx "${VER}" && echo "✅ tag ${VER}" \
    || { echo "❌ no local tag ${VER}"; exit 1; }

# 1.3 tests green — FIX #1: this is a real GATE now (was print-only).
if bash tests/run_all.sh > /tmp/{{PROJECT}}_test.log 2>&1; then
    tail -3 /tmp/{{PROJECT}}_test.log
    echo "✅ tests all green"
else
    tail -20 /tmp/{{PROJECT}}_test.log
    echo "❌ tests not green (runner exited non-zero) — aborting"; exit 1
fi

# 1.4 credential scan (guardrail 1)
bash scripts/scan-credentials.sh "$RELEASE_DIR" \
    && echo "✅ credential scan clean" \
    || { echo "❌ credential leak, aborting"; exit 1; }

# 1.5 gh CLI authenticated
gh auth status 2>&1 | grep -q "Logged in to github.com" \
    && echo "✅ gh logged in" \
    || { echo "❌ run gh auth login"; exit 1; }
```

**Any failure → stop, fix, restart Phase 1.**

## Phase 5. Commit + tag

```bash
cd "$PUBLISH_WT"
git add -A

# 5.2 staged-diff scan (guardrail 3) — FIX #5: scan for the ORIGINAL private
# values + secrets + dev paths. (codex audit: never scan for the redaction
# *placeholder* names like <private-port> — Phase 4.5 deliberately writes
# those, so matching them aborts a correctly-redacted publish. Add your
# project's real private port/org/ssh-key literals here if you want them
# caught too — but never the angle-bracket placeholders.)
LEAK=$(git diff --staged | grep -icE \
  'ghp_[A-Za-z0-9]{36}|github_pat_[A-Za-z0-9_]{82,}|password[[:space:]]*=|api[_-]?key[[:space:]]*=|{{PRIVATE_GIT_HOST}}|/Users/[a-z]+/|/home/[a-z]+/' \
  2>/dev/null || true)
[ "$LEAK" -eq 0 ] && echo "✅ staged diff clean" \
    || { echo "❌ staged diff has $LEAK leak suspicions, stop"; exit 1; }

git commit -m "release: {{PROJECT}} ${VER}

Clean release snapshot at tag ${VER} — no WIP history.
Source-of-truth: private dev tree (not exposed)."
git tag -a "${VER}" -m "{{PROJECT}} ${VER}"

# 5.5 🔴 author identity check before push
AUTHOR=$(git log -1 --format="%an <%ae>" HEAD)
[ "$AUTHOR" = "{{PROJECT}} <noreply@github.com>" ] \
    && echo "✅ author OK" \
    || { echo "❌ author=$AUTHOR — fix git config, git commit --amend --reset-author --no-edit, re-tag"; exit 1; }
cd ..
```

## Phase 6. Push to GitHub

```bash
cd "$PUBLISH_WT"

# 6.0 🔴 temporary token-in-URL (gh CLI token is not used by git push).
# FIX #4 (codex audit): derive the token — Phase 1 only checks `gh auth
# status`, so GH_TOKEN is normally NOT exported; assuming it yields a broken
# `https://x-access-token:@github.com/...` URL. Fall back to `gh auth token`.
GH_TOKEN="${GH_TOKEN:-$(gh auth token 2>/dev/null)}"
[ -n "$GH_TOKEN" ] || { echo "❌ no GitHub token (set GH_TOKEN or run gh auth login)"; exit 1; }
ORIG_URL=$(git remote get-url origin)
git remote set-url origin "https://x-access-token:${GH_TOKEN}@github.com/${REPO}.git"
trap 'git remote set-url origin "$ORIG_URL" 2>/dev/null || true' EXIT

# 6.1/6.2 push main + tag with --force (publish-wt is per-release wipe-rewrite;
#         this script solely owns main, force is the design intent;
#         --force-with-lease fails on a fresh worktree with "stale info").
# FIX #2 (codex audit): do NOT pipe the main push through `| tail` — that
# masks its exit code and the SOP would tag/release a stale main. Abort
# explicitly on a failed main push BEFORE pushing the tag.
if ! git push origin main --force; then
    echo "❌ main push failed — NOT tagging (prevents stale-main release)"; exit 1
fi
git push origin "${VER}" --force

# 6.3 restore remote URL (strip token) + clear trap
git remote set-url origin "$ORIG_URL"
trap - EXIT

# 6.4 FIX #2: explicit idempotent token-scrub verification — do NOT rely on
# the trap alone (it does not fire if phases run in separate shells).
git remote get-url origin | grep -q 'x-access-token' \
    && { echo "❌ token still in remote URL — manual scrub required"; exit 1; } \
    || echo "✅ remote URL token-free"

# 6.5 verify visible on GitHub
gh api "repos/${REPO}/tags" -q ".[] | select(.name==\"${VER}\") | .name"
# ✅ expect: ${VER}
cd ..
```

## Phase 7. Create GitHub Release

```bash
cd "<project-root>"

# 7.1 assets (NOT the source tarball — it carries un-redacted dev narrative)
ASSETS=(
    "${RELEASE_DIR}/{{PROJECT}}-${VER}.skill"
    "${RELEASE_DIR}/checksums.txt"
    "${RELEASE_DIR}/sbom.cyclonedx.json"
)
for f in "${ASSETS[@]}"; do
    [ -f "$f" ] && echo "✅ $f" || { echo "❌ $f missing"; exit 1; }
done

# 7.1b FIX #4: verify artifact integrity against checksums.txt BEFORE publish.
( cd "$RELEASE_DIR" && shasum -a 256 -c checksums.txt ) \
    && echo "✅ checksums verified" \
    || { echo "❌ checksum mismatch — aborting publish"; exit 1; }

# 7.2 idempotency — FIX #6: if release exists, the tag/commit are already
# pushed; do NOT just exit. Re-run is clean via --clobber or rollback B.
if gh release view "${VER}" --repo "$REPO" >/dev/null 2>&1; then
    echo "⚠ Release ${VER} exists. To re-publish: gh release delete ${VER} --repo $REPO --yes (rollback B), then re-run Phase 7."
    exit 1
fi

# 7.3 create draft → human preview → promote
gh release create "${VER}" --repo "$REPO" \
    --title "{{PROJECT}} ${VER}" \
    --notes-file "${RELEASE_DIR}/RELEASE_NOTES.md" \
    --target main --draft "${ASSETS[@]}"
gh release view "${VER}" --repo "$REPO" --web
# 🔍 HUMAN review: title / notes / assets / tag — then confirm:
gh release edit "${VER}" --repo "$REPO" --draft=false --latest
```

## 🚨 Rollback

- **A. tag pushed, no release**: `git tag -d ${VER}; git push origin :refs/tags/${VER}` then fix + redo.
- **B. release wrong**: `gh release delete ${VER} --repo $REPO --yes` or `gh release upload ${VER} <asset> --clobber`.
- **C. main commit wrong**: just re-run Phase 3–6 (per-release force-rewrite; no history concept).
- **D. secret leaked to remote**: (1) **revoke** the token immediately; (2) FIX #8: re-run Phase 3–6 to force-rewrite a clean snapshot (preferred — consistent with this SOP's wipe-rewrite model; `git filter-repo` is optional and only if it is installed); (3) notify cloners to re-clone; (4) check the private remote leaked too.
