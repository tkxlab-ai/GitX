# GitHub Release SOP — {{PROJECT}}

> **Generated by** `gitx-sop` (gitx-release {{GITX_VERSION}}) on {{DATE}}.
> **Trigger**: dev says `推 GitHub` / `publish vX.Y.Z` → the project's AI follows
> these **8 Phases** in order.
>
> **Principles**
> - `gitx-release` produces release artifacts into `Release/{{PROJECT}}-vX.Y.Z/` (untouched).
> - This SOP only takes over "push the release to GitHub".
> - One independent commit + tag per version — clean snapshot, no WIP history.
> - The private Git host **never** appears in any GitHub metadata (URL / author / config).
> - **This document is a runbook for a human-supervised AI. It does NOT auto-run.
>   Every `git push` / `gh release` requires explicit human confirmation.**

## 0. Repo topology

| Layer | Location | Purpose | git |
|----|------|------|----------|
| Private main | project root | dev + HANDOFF + tests | private remote (no public exposure) |
| Publish worktree | `<root>/.github-publish-wt/` | GitHub snapshot (per-release rewrite) | **own .git**, origin = `https://github.com/{{REPO}}.git` |
| Release artifacts | `<root>/Release/{{PROJECT}}-vX.Y.Z/` | gitx-release output (.skill + tarball + SBOM) | not in git |

**Key facts**
1. GitHub source-of-truth = the sanitized public snapshot (Phase 4 redaction MUST pass).
2. Publish worktree is wiped + rebuilt every release (no cross-version history).
3. The private remote is **never** added to the publish worktree's `.git/config`.

```bash
    && echo "✅ credential scan clean" \
    || { echo "❌ credential leak, aborting"; exit 1; }

# 1.5 push auth available — gh auth OR GH_TOKEN. (v1.7.8 codex stop-gate:
# do NOT hard-require gh here, or Phase 6's no-gh token-in-URL fallback
```

```bash
cd "<project-root>"

# 4.1 extract the gitx-release source tarball (already sanitize-ignore filtered)
tar xzf "${RELEASE_DIR}/{{PROJECT}}-${VER}-source.tar.gz" \
    -C "$PUBLISH_WT" --strip-components=1

# 4.2 add public-repo-only meta files

cp CODE_OF_CONDUCT.md "$PUBLISH_WT/" 2>/dev/null || true
cp -r .github         "$PUBLISH_WT/" 2>/dev/null || true

# 4.3 verify key files present — FIX #3: missing key file now ABORTS.
cd "$PUBLISH_WT"
missing=0
for f in README.md LICENSE CHANGELOG.md INSTALL.md install.sh; do
    [ -e "$f" ] && echo "✅ $f" || { echo "❌ $f missing"; missing=1; }
done
[ "$missing" -eq 0 ] || { echo "❌ extraction incomplete — aborting"; exit 1; }
cd ..

# 4.4 defensive credential re-scan (worktree composed)
bash scripts/scan-credentials.sh "$PUBLISH_WT" \
    && echo "✅ worktree scan clean" \
    || { echo "❌ worktree credential leak"; exit 1; }

# 4.5 🔴 public redaction (project-specific — fill the table below)
#   Delete dev-only files (transfer guides / session notes / pre-vN backups)
#   then sed-replace private host/port/org/ssh-key/dev-paths → placeholders.
#   Verification grep — any remaining hit → exit 1.
# v1.7.4 portability: most projects do NOT have release-sanitize-public.sh
# (gitx-release itself doesn't). Detect it; otherwise use a generic
# redaction fallback. A MANDATORY post-redaction verification grep runs
# regardless of which path executed — fail-closed.
if [ -x scripts/release-sanitize-public.sh ]; then
    bash scripts/release-sanitize-public.sh "$PUBLISH_WT" \
        && echo "✅ project redaction done" \
        || { echo "❌ redaction failed"; exit 1; }
else
    echo "⚠ no scripts/release-sanitize-public.sh — generic redaction fallback"
    # delete common dev-only files
    rm -f "$PUBLISH_WT"/HANDOFF.md "$PUBLISH_WT"/HANDOFF.archive.md \
          "$PUBLISH_WT"/REVIEW.md "$PUBLISH_WT"/NEXT_SESSION_NOTES.md 2>/dev/null || true
    rm -rf "$PUBLISH_WT"/.memory "$PUBLISH_WT"/.gitx 2>/dev/null || true
    # sed-redact the known private markers in-place
    grep -rIlZ -e '{{PRIVATE_GIT_HOST}}' "$PUBLISH_WT" 2>/dev/null \
      | xargs -0 -r sed -i.bak "s#{{PRIVATE_GIT_HOST}}#<private-git-host>#g" 2>/dev/null || true
    find "$PUBLISH_WT" -name '*.bak' -delete 2>/dev/null || true
fi
# MANDATORY verification grep — runs after EITHER path; any residual
# private marker (host / dev-home / private-org slug) aborts the publish.
if grep -rIqE '{{PRIVATE_GIT_HOST}}|/Users/[a-z]+/|/home/[a-z]+/|<private-(port|org|ssh-key)>' "$PUBLISH_WT" 2>/dev/null; then
    echo "❌ residual private marker after redaction — aborting publish"; exit 1
fi
echo "✅ redaction verified clean"
```

> `/Users/<dev>/` and `/home/<dev>/` → `<dev-home>/` · dev hostname → `<dev-host>`.

## Phase 5. Commit + tag

```bash
cd "$PUBLISH_WT"
git add -A

# 5.2 staged-diff scan (guardrail 3) — FIX #5: scan for the ORIGINAL private
# values + secrets + dev paths. (codex audit: never scan for the redaction
# *placeholder* names like <private-port> — Phase 4.5 deliberately writes
# those, so matching them aborts a correctly-redacted publish. Add your

  'ghp_[A-Za-z0-9]{36}|github_pat_[A-Za-z0-9_]{82,}|password[[:space:]]*=|api[_-]?key[[:space:]]*=|{{PRIVATE_GIT_HOST}}|/Users/[a-z]+/|/home/[a-z]+/' \
```

## Phase 6. Push to GitHub

```bash
cd "$PUBLISH_WT"

# 6.0 🔴 v1.7.7: PREFER the gh credential helper — no token ever touches
# the remote URL or .git/config. If `gh` is installed and authenticated
# (browser OAuth via `gh auth login`), `gh auth setup-git` makes git push
# authenticate through gh's keyring transparently. This is strictly safer
# than token-in-URL (no plaintext PAT anywhere, nothing to scrub). The
# token-in-URL path survives ONLY as an explicit no-gh fallback.
USED_TOKEN_URL=0
if command -v gh >/dev/null 2>&1 && gh auth status >/dev/null 2>&1; then
    gh auth setup-git
    echo "✅ using gh credential helper (no token-in-URL, nothing to scrub)"
else
    # fallback: no gh — temporary token-in-URL (FIX #4), scrubbed after push.
    GH_TOKEN="${GH_TOKEN:-}"
    [ -n "$GH_TOKEN" ] || { echo "❌ no gh and no GH_TOKEN — run gh auth login (preferred) or export GH_TOKEN"; exit 1; }
    ORIG_URL=$(git remote get-url origin)
    git remote set-url origin "https://x-access-token:${GH_TOKEN}@github.com/${REPO}.git"
    trap 'git remote set-url origin "$ORIG_URL" 2>/dev/null || true' EXIT
    USED_TOKEN_URL=1
fi

# 6.1/6.2 push main + tag with --force (publish-wt is per-release wipe-rewrite;
#         this script solely owns main, force is the design intent;
#         --force-with-lease fails on a fresh worktree with "stale info").
# FIX #2 (codex audit): do NOT pipe the main push through `| tail` — that
# masks its exit code and the SOP would tag/release a stale main. Abort
# explicitly on a failed main push BEFORE pushing the tag.
if ! git push origin main --force; then
    echo "❌ main push failed — NOT tagging (prevents stale-main release)"; exit 1
fi
git push origin "${VER}" --force

# 6.3 + 6.4: only relevant on the no-gh fallback path. gh-helper path
# never put a token in the URL, so there is nothing to restore/scrub.
if [ "$USED_TOKEN_URL" -eq 1 ]; then
    git remote set-url origin "$ORIG_URL"   # strip token
    trap - EXIT
    # FIX #2: explicit idempotent scrub verification — do NOT rely on the
    # trap alone (it does not fire if phases run in separate shells).
    git remote get-url origin | grep -q 'x-access-token' \
        && { echo "❌ token still in remote URL — manual scrub required"; exit 1; } \
        || echo "✅ remote URL token-free"
else
    echo "✅ gh-helper path: no token was ever in the URL"
fi

# 6.5 verify visible on GitHub
gh api "repos/${REPO}/tags" -q ".[] | select(.name==\"${VER}\") | .name"
# ✅ expect: ${VER}
cd ..
```

## Phase 7. Create GitHub Release

```bash
cd "<project-root>"

# 7.1 assets (NOT the source tarball — it carries un-redacted dev narrative)
ASSETS=(
    "${RELEASE_DIR}/{{PROJECT}}-${VER}.skill"
    "${RELEASE_DIR}/checksums.txt"
    "${RELEASE_DIR}/sbom.cyclonedx.json"
)
for f in "${ASSETS[@]}"; do
    [ -f "$f" ] && echo "✅ $f" || { echo "❌ $f missing"; exit 1; }
done

# 7.1b FIX #4: verify artifact integrity against checksums.txt BEFORE publish.
( cd "$RELEASE_DIR" && shasum -a 256 -c checksums.txt ) \
    && echo "✅ checksums verified" \
    || { echo "❌ checksum mismatch — aborting publish"; exit 1; }

# 7.2 idempotency — FIX #6: if release exists, the tag/commit are already
# pushed; do NOT just exit. Re-run is clean via --clobber or rollback B.
if gh release view "${VER}" --repo "$REPO" >/dev/null 2>&1; then
    echo "⚠ Release ${VER} exists. To re-publish: gh release delete ${VER} --repo $REPO --yes (rollback B), then re-run Phase 7."
    exit 1
fi

# 7.3 create draft → human preview → promote
gh release create "${VER}" --repo "$REPO" \
    --title "{{PROJECT}} ${VER}" \
    --notes-file "${RELEASE_DIR}/RELEASE_NOTES.md" \
    --target main --draft "${ASSETS[@]}"
gh release view "${VER}" --repo "$REPO" --web
# 🔍 HUMAN review: title / notes / assets / tag — then confirm:
gh release edit "${VER}" --repo "$REPO" --draft=false --latest

# 7.6 v1.7.4: when backfilling older releases AFTER a newer one already
# exists, `gh release create` silently steals "Latest" (it marks the
# most-recently-created release latest). Always re-assert latest on the
# highest semver afterwards:
NEWEST=$(gh release list --repo "$REPO" --json tagName -q '.[].tagName' \
    | sort -V | tail -1)
gh release edit "$NEWEST" --repo "$REPO" --latest
echo "✅ latest re-asserted on $NEWEST"
```

## Phase 8. Wrap-up

```bash
# 8.1 update HANDOFF Current State + Dev Log (via /handoff; SOP does not edit)
# 8.2 append a row to Appendix A of this SOP
# 8.3 push private remote if SOP/HANDOFF changed (human-confirmed)

# 8.4 v1.7.4 SOP completeness gate — every pushed tag MUST have a
# matching GitHub Release (a tag with no Release means Phase 7 was
# skipped; the GitX self-publish session skipped it for 3 versions).
_missing=0
for _t in $(gh api "repos/$REPO/tags" -q '.[].name' 2>/dev/null); do
    if ! gh release view "$_t" --repo "$REPO" >/dev/null 2>&1; then
        echo "❌ tag $_t has no GitHub Release — Phase 7 incomplete"; _missing=1
    fi
done
[ "$_missing" -eq 0 ] && echo "✅ every tag has a GitHub Release" \
    || { echo "❌ SOP incomplete: backfill missing Releases before sign-off"; exit 1; }
```

## 🚨 Rollback

- **A. tag pushed, no release**: `git tag -d ${VER}; git push origin :refs/tags/${VER}` then fix + redo.
- **B. release wrong**: `gh release delete ${VER} --repo $REPO --yes` or `gh release upload ${VER} <asset> --clobber`.
- **C. main commit wrong**: just re-run Phase 3–6 (per-release force-rewrite; no history concept).
- **D. secret leaked to remote**: (1) **revoke** the token immediately; (2) FIX #8: re-run Phase 3–6 to force-rewrite a clean snapshot (preferred — consistent with this SOP's wipe-rewrite model; `git filter-repo` is optional and only if it is installed); (2b) **v1.7.4: force-rewriting `main` does NOT clean a tag built from the leaked snapshot — that tag still points at the bad blob. Delete/repoint every such tag: `git push origin :refs/tags/<ver>` and `gh api -X DELETE repos/$REPO/git/refs/tags/<ver>`, then re-tag from the clean commit.** (3) notify cloners to re-clone; (4) check the private remote leaked too.
